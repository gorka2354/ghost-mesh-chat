# Ghost Mesh Chat — Инкременты

План разработки по принципу: **каждый инкремент = работающая версия**.

---

## Инкремент 1: Минимальный P2P чат (1 на 1)

**Цель:** Два браузера обмениваются текстовыми сообщениями напрямую.

**Задачи:**
- [ ] Инициализировать проект (HTML + CSS + JS)
- [ ] Подключить PeerJS (CDN)
- [ ] При открытии — генерировать случайный Peer ID
- [ ] UI: поле ввода ID собеседника + кнопка "Подключиться"
- [ ] Установить WebRTC DataChannel между двумя пирами
- [ ] Отправка/получение текстовых сообщений
- [ ] Минимальный UI: окно чата + поле ввода

**Результат:** Открываешь в двух вкладках → вводишь ID → пишешь → работает.

**Проверка:** Открыть на двух устройствах в одной сети, отправить сообщение.

---

## Инкремент 2: Комнаты и ссылки-приглашения

**Цель:** Создать "комнату" и пригласить по ссылке (без ручного ввода ID).

**Задачи:**
- [ ] Генерация уникального room ID
- [ ] URL формата `site.com/#room-abc123`
- [ ] При открытии ссылки — автоматическое подключение к создателю комнаты
- [ ] Отображение кто онлайн в комнате
- [ ] Кнопка "Скопировать ссылку"

**Результат:** Создал комнату → скопировал ссылку → отправил другу → он открыл → вы в чате.

---

## Инкремент 3: Mesh-сеть (групповой чат)

**Цель:** 3+ участника, каждый подключён к каждому.

**Задачи:**
- [ ] При подключении нового пира — он получает список существующих пиров
- [ ] Новый пир устанавливает DataChannel с каждым существующим
- [ ] Broadcast сообщений: отправка всем подключённым пирам
- [ ] Обработка отключения пира (удаление из списка)
- [ ] Отображение количества участников
- [ ] Никнеймы (случайные или выбранные)

**Результат:** Комната на 3-10 человек, все видят все сообщения.

---

## Инкремент 4: UI/UX и тёмная тема

**Цель:** Приятный интерфейс, похожий на мессенджер.

**Задачи:**
- [ ] Дизайн в стиле "хакерский мессенджер" (тёмная тема, моноширинный шрифт)
- [ ] Аватарки по первой букве никнейма (цветные кружки)
- [ ] Время отправки сообщения
- [ ] Анимация появления сообщений
- [ ] Звук уведомления при новом сообщении
- [ ] Адаптивная вёрстка (мобилка + десктоп)
- [ ] Индикатор "печатает..."

**Результат:** Выглядит как настоящий мессенджер, приятно пользоваться.

---

## Инкремент 5: PWA — установка как приложение

**Цель:** Пользователь может "установить" чат на телефон/десктоп.

**Задачи:**
- [ ] Создать `manifest.json` (название, иконки, цвета, display: standalone)
- [ ] Нарисовать иконку приложения (или сгенерировать)
- [ ] Написать `service-worker.js` (кэширование assets для офлайн)
- [ ] Splash screen при загрузке
- [ ] Мета-теги для iOS (apple-mobile-web-app-capable)
- [ ] Проверить установку на Android (Chrome) и iOS (Safari)

**Результат:** Иконка на домашнем экране → открывается как приложение → полноэкранный чат.

---

## Инкремент 6: Передача файлов

**Цель:** Drag & drop файлов прямо в чат.

**Задачи:**
- [ ] Drag & drop зона в окне чата
- [ ] Разбивка файла на чанки (DataChannel имеет лимит ~16KB на сообщение)
- [ ] Прогресс-бар отправки/получения
- [ ] Превью для изображений
- [ ] Скачивание полученного файла
- [ ] Лимит размера (например, 50 МБ)

**Результат:** Перетащил файл → отправилось всем в комнате → скачали.

---

## Инкремент 7: Шифрование сообщений (E2EE)

**Цель:** Дополнительный слой шифрования поверх WebRTC.

**Задачи:**
- [ ] Генерация пары ключей (Web Crypto API) при создании комнаты
- [ ] Обмен публичными ключами при подключении
- [ ] Шифрование каждого сообщения перед отправкой (AES-GCM)
- [ ] Отображение "замочка" в UI
- [ ] Верификация ключей (показать fingerprint для сравнения)

**Результат:** Даже если перехватить трафик — сообщения не прочитать.

---

## Инкремент 8: Деплой и публикация

**Цель:** Проект доступен по URL, можно показать людям.

**Задачи:**
- [ ] Деплой на GitHub Pages / Vercel / Netlify (бесплатно)
- [ ] Красивый домен (опционально)
- [ ] README на GitHub с описанием и скриншотами
- [ ] Open Graph мета-теги (превью при шаринге ссылки)
- [ ] Лендинг-страница: что это, как работает, кнопка "Создать чат"

**Результат:** Рабочий продукт с URL, который можно отправить кому угодно.

## Бонусные инкременты (после MVP)

### 9. Голосовые сообщения
- Запись через MediaRecorder API → отправка через DataChannel

### 10. Свой signaling-сервер
- WebSocket-сервер на Node.js → независимость от PeerJS Cloud
- Деплой на Fly.io / Railway (бесплатный tier)

### 11. Tauri (десктоп-приложение)
- Обёртка в нативное приложение (~5 МБ)
- Сборка для Windows / macOS / Linux

### 12. Capacitor (мобильное приложение)
- Обёртка для Android / iOS
- Публикация в Google Play (F-Droid для анонимности)

---

## Правила работы

1. **Один инкремент = один PR** (или серия коммитов)
2. **Каждый инкремент проверяется вручную** перед переходом к следующему
3. **Не забегать вперёд** — сначала работающий чат 1-на-1, потом фичи
4. **Коммит после каждой задачи** — не копить изменения
5. **Тестировать на телефоне** после каждого инкремента
